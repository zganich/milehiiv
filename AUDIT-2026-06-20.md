## TL;DR

**The site is live but is NOT a working product.** The backend APIs are real;
the front-end is a static mockup disconnected from them. Checkout has been
**paused** so no one can pay for something that can't deliver.

## What actually works

- Marketing pages (live at milehiiv.com, valid SSL).
- Backend API routes are genuinely implemented:
  - `/api/auth/*` (register/login/me — bcrypt + JWT)
  - `/api/import/*` (real Uber/Lyft CSV parsers, ~300 lines each)
  - `/api/mileage/{summary,deadhead,trips,gaps}` (real logic)
  - automatic gap detection (`detectMileageGaps()`)
  - `/api/checkout` + `/api/stripe/webhook` (Stripe)
- **Fixed 2026-06-20:** year-aware IRS rate table (`lib/irs-rates.ts`) — was
  hardcoded to 2024 $0.67, wrong for every 2025+ figure. CSV-only days now
  surface an honest, clearly-labeled *potential* deadhead estimate instead of a
  silent 0.

## What does NOT work (the real 0→1 gap)

The front-end is not wired to the backend. Specifically:

1. **No account flow** — there is *no login or register page*. A user cannot
   create an account at all.
2. **Upload does nothing** — `UploadZone.tsx` is a static `<div>`; no file
   input, no POST to `/api/import`.
3. **Dashboard shows fake data** — `dashboard/page.tsx` has hardcoded numbers
   ("14,320 miles", "3,180 deadhead"); it never calls the mileage APIs.
4. **No report** — `dashboard/reports/page.tsx` is a 14-line stub. There is no
   PDF/CSV generation anywhere. The "Schedule C / CPA-ready report" promise is
   marketing copy, not a feature.
5. No client-side session/token storage anywhere; only 2 interactive
   components in the whole app.

**Net:** a visitor could pay (Stripe was wired) but then had no way to register,
upload, or get anything back. That's why checkout is now paused.

## Checkout is PAUSED (default-off)

Gated behind `CHECKOUT_ENABLED` (env, defaults OFF):
- Pricing UI shows disabled "Coming soon" buttons + in-development banner.
- `/api/checkout` refuses POST → `303 /pricing?checkout=paused` (no Stripe
  session), so a direct request can't bypass the disabled button.

## Relaunch path (the real 0→1, in order)

1. Auth UI: register + login pages, store JWT, session guard on `/dashboard`.
2. Wire `UploadZone` → `/api/import` (with auth header).
3. Make `/dashboard` fetch `/api/mileage/summary` + `/deadhead` (kill hardcoded numbers).
4. Build the actual report: a `/api/reports/export` (CSV, per-trip log +
   Schedule C summary) + a real reports page (download + print-to-PDF).
5. QA the full loop with real Uber/Lyft CSV test data; verify deadhead +
   deduction math against the IRS rate table.
6. Only then: set `CHECKOUT_ENABLED=true` in Vercel and resume GTM.
